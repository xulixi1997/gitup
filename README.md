核心功能：将手机通讯录中的常用联系人，与一个特定的待办事项类型绑定。例如，绑定“妈妈”为“每周打电话”，绑定“房东”为“交房租”。点击联系人头像，直接创建一个带有预设标题和倒计时的待办事项。

特色：将抽象的待办事项与具体的人关联起来，简化了重复性社交任务的创建流程。
1. 产品概述
应用名称： QuickContactTask

产品口号： 为人际关系，设个提醒。

核心价值： 通过手动创建联系人与任务的绑定，实现一键快速生成待办事项，专注于简化基于人际关系的重复性任务管理。

目标用户： 注重生活条理，希望维护规律性社交互动（如家人问候、定期付款）的用户。

2. 主题颜色定义
我们采用一种温暖、沉稳的“暮色森林”主题，营造一种安静、专注的氛围。

主色调： #3E2723 (深褐色)

辅助色： #5D4037 (咖啡棕)

强调色： #FF9800 (暖橙色) - 用于创建按钮、重要状态，增添活力。

背景色： #261C19 (近乎黑色的深棕)

表面色： #3E2723 (与主色相同，用于卡片、底部栏)

文字色：

主要文字：#FFFFFF (白色)

次要文字：#BCAAA4 (浅褐色)

禁用文字：#8D6E63 (灰褐色)

3. 信息架构与底部导航
底部导航栏包含四个核心页面：

首页： 展示即将到期的任务列表。

设置： 应用偏好管理。

5. 核心交互流程 (手动创建版)
流程一：手动创建联系人及其绑定任务
用户在 「联系人」 页面点击右上角的 +。

跳转到 「添加联系人」 页面：

头像： 允许从相册选择或使用默认头像（显示姓名首字母）。

姓名： 输入框，必填。

绑定任务：

任务标题： 输入框（如：“每周打电话”）。

任务周期： 选择器（无重复、每天、每周、每月、每年等）。

点击“保存”，一个新的联系人及其任务模板就创建完成了。

流程二：通过联系人快速创建任务实例
用户在 「联系人」 页面，点击已存在的“妈妈”列表项。

应用立即根据绑定的模板（“每周打电话”），在后台自动生成一个任务实例。

系统给出一个轻量级的视觉反馈（如一个“任务已创建！”的Snackbar提示），用户停留在当前页面。

新生成的任务会立刻出现在 「首页」 和 「任务」 页面的“待完成”列表中。

流程三：标记任务完成
用户在 「首页」 点击卡片上的“完成”按钮，或在 「任务」 页面向左滑动并点击“完成”。

该任务被标记为“已完成”，并从待办列表中移除。

系统行为： 如果该任务有关联的周期，系统会自动计算下一个到期日，并生成一个新的待办事项。例如，一个“每周”任务在10月26日完成后，会自动创建一个到期日为11月2日的新任务。
